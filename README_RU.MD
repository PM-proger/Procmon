
# Функционал

Программа разработана в рамках выполнения тестового задания и реализует следующие функции:

1) Отображает окно с автоматически обновляемым актуальным списком запущенных процессов Windows.

2) Позволяет завершить выбранный процесс.

3) Позволяет повысить привилегии до уровня администратора (без таких прав перечень доступных процессов значительно ограничен).

4) Позволяет получить список модулей процесса и отправить его в зашифрованном виде на сервер (*).

(*) Реальный адрес сервера в проекте на GITHUB не раскрывается.
Если Ваша компания "опознала" свое тестовое задание, Вы можете указать реальный IP-адрес сервера и программа должна работать правильно, либо определить его при
сборке проекта ('#define SERVER_IP' в главной программе - ProcMon4.cpp)


5) Позволяет получить от сервера ранее направленные ему данные, расшифровать и отобразить их на экране.

6) Распознает попытку повторного запуска программы и не позволяет ее выполнить.

7) Содержит ограничения по минимальноу размеру основного окна и не позволяет пользователю уменьшить окно меньше минимально допустимых размеров.

8) Позволяет свернуть программу в трей, а находясь в трее - либо восстановиться до полного окна, либо завершить работу (локальное меню вызывается нажатием правой кнопкой мыши по иконке в трее).


# Техническая реализация

## Сборка проекта

Реализована CMAKE-версия проекта, разработка осуществлялась в Visual Studio 2019. 

Реальный адрес сервера в проекте на GITHUB не раскрывается.
Если Ваша компания "опознала" свое тестовое задание, Вы можете указать реальный IP-адрес сервера и программа должна работать правильно.
Тестирование на реальном сервере показало полную работоспособность сетевой части.

## Основные модули

### ProcMon4 - основная программа

Procmon4.cpp - основная программа,

Job_SendDLL.cpp/.h - функция отправки данных на сервер,

Job_RecvDLL.cpp/.h - функция получения данных с сервера.

### UtilsProc_LIB - вспомогательные библиотеки

utilsnet_lib.cpp/.h - библиотека сетевых функций,

utilsproc_lib.cpp/.h - библиотека работы с процессами Windows,

utilswin_lib.cpp/.h - библиотека разных процедур для работы с WINAPI (работа с треем, информационные сообщения, повышение привилегий до админа),

utilsmisc_lib.cpp/.h - работа с JSON, шифрование, прочие вспомогательные функции

## Интерфейс

Графический интерфейс выполнен средствами WINAPI (MFC или иные аналогичные фреймворки не используются).

Список процессов отображается в элементе ListView, который обновляется по событиям таймера.
Сетевой функционал (обмен с сервером) реализован в отдельном потоке (std::thread) средствами winsock2.

Шифрование осуществляется простейшим шифром без привлечения сторонних библиотек.


## Протокол обмена с сервером

### Общая информация

URL сервера: http://XXX.XXX.XXX.XXX/p/applicants.php
(реальный адрес сервера здесь не раскрывается)

Content-Type: application/json

### Отправка данных

Метод запроса: POST.

JSON-формат отправляемых данных:

{“cmd”: 1, “rid”: “%УНИКАЛЬНЫЙ_ИДЕНТИФИКАТОР%”, “data”: “%ШИФРОВАННАЯ_СТРОКА%”}
Где 
cmd – команда, где 1 – отправить данные, 2 – получить данные
rid – уникальный идентификатор запроса
data -	 шифрованная строка

Response:
{“rid”: “%УНИКАЛЬНЫЙ_ИДЕНТИФИКАТОР%”, “status”: “true”}
Где 
status – ответ сервера об успешном получении данных
rid – идентификатор запроса, используемый при отправке данных


### Получение данных
Send request POST Data:
{“cmd”: 2, “rid”: “%УНИКАЛЬНЫЙ_ИДЕНТИФИКАТОР%”}
Где 
cmd – команда, где 1 – отправить данные, 2 – получить данные
rid – идентификатор запроса, используемый при отправке данных

Response:
{“rid”: “%УНИКАЛЬНЫЙ_ИДЕНТИФИКАТОР%”, “data”: “%ШИФРОВАННАЯ_СТРОКА%”}
Где 
rid – идентификатор запроса, используемый при отправке данных
data – шифрованная строка
